FORMULÁRIOS

O elemento <form> especifica um formulário em HTML que permite a entrada de dados por parte do usuário. Funciona como um contêiner para diversos outros elementos de entrada, como campo de texto, caixas de seleção, botões, etc.


<form> campos de entrada </form>


Atributo method

O atributo method define o método de envio dos dados, podendo ser por GET (dados através da URL) ou POST (é enviado no corpo da solicitação). O método de envio padrão é o GET.

Método de Envio GET

Quando usamos o método de envio GET, todos os dados do formulário são enviados através de uma string dentro da própria URL do site. Onde os dados são adicionados usando-se um par chave=valor. DEVE SER USADO APENAS PARA DADOS NÃO CONFIDÊNCIAIS

http://www.servidor.com.br/userform.php?username=Pedro

Assim, essa solicitação executaria o script script.php em php no diretório raiz do servidor e passaria o valor Pedro para a variável username. Os caracteres abaixo são usados para separar os conteúdos na URL.

  • ? : Separa a localização do dado
  • & : separa os pares valor=chave
  • + : Substitui o caracter espaço.

Vantagens: É simples e fácil de trabalhar.

Desvantagens: Os dados ficam visíveis e há uma limitação de 2048 caracteres na URL.

Método de Envio POST

Quando usamos o método de envio POST, todos os dados do formulário são enviados por transação HTTP, basicamente, as solicitações que usam post enviam os dados do formulário como um pacote no corpo da solicitação.

Vantagens: Não Possui limitação de tamanho de dados enviados e os dados não ficam visíveis

Desvantagens: Os envios não podem ser marcados.



Atributo action

O atributo action define a ação que deve ser feita após o envio de um formulário, onde normalmente, os dados são enviados para um arquivo no servidor, após a interação com o usuário. Caso o valor de action seja omitido, os dados são enviados para a página atual.



Atributo target

O atributo target especifica para onde exibir a resposta recebida após o envio do formulário. Ou seja, se quer que seja aberta uma nova aba, recarrega a página atual, abra uma nova página sobre a atual, dentre outras formas.

  • _black: A resposta é exibida em uma nova janela ou aba.
  • _self (padrão): A resposta é exibida na janela atual.
  • _parent: A resposta é exibida no quadro pai.
  • _top: A resposta é exibida em todo o corpo da janela./li>
  • framename (nome de um iframe): A reposta é exibida no iframe citado.


Atributo autocomplete

O atributo autocomplete especifica se o formulário deve ter preenchimento automático ativado, através do valor on. Após a sua ativação, o navegador irá completar os valores automaticamente com base no histórico dos dados preenchidos.



Atributo novalidate

O novalidate é um atributo booleano. Quando presente, especifica que os dados de entrada do formulário não devem ser validados ao serem enviados

Assim, podemos ver a sintaxe com todos os atributos importantes do elemento <form>

<form action="/action_page.php" target="_blank" method="post" autocomplete="on" novalidate>
Dados de entrada
</form>

Resultando, que os atributos, caso omitidos, serão aplicados os valores padrões.



ELEMENTOS DE FORMULÁRIOS

Elemento label

O elemento <label> define um rótulo para diversos elementos de um formulário, como uma descrição do valor que deve ser inserido: Como nome, idade, sexo, dentre outras possibilidades.


<label for="id"> Rótulo </label>

Labels são úteis pois facilitam tanto na interação com o usuário, pois fazem parte os elementos clicáveis, quanto para leitores de telas enfatizarem os elementos de entrada.

Para fazer essa ligação entre o label e o elemento de entrada, deve usar um id único para o elemento de entrada. Esse id deve ser usado como valor do atributo for dentro da tag inicial do elemento <label for="id">.



Elemento input

É o elemento mais utilizado, que pode assumir diversas funcionalidades dependendo do valor do atributo type especificado na tag inicial. Os elementos <input type=""> são elementos vazios, que não precisam de uma tag de fechamento.

input atributos

  • value: O atributo value especifica um valor inicial para um campo de entrada.
  • readonly: Especifica que um campo de entrada é somente de leitura, ou seja, não pode ser mais modificado, mas é enviado ao enviar o formulário.
  • disabled: Especifica que um campo de entrada deve ser desativado, se tornando inutilizável e não podendo sem clicável, além disso, o dado não é enviado junto com o formulário.
  • size: Especifica a largura visível, em caracteres, de um campo de entrada, sendo por padrão o valor 20. Funciona em (input text, search, tel, url, email e password).
  • maxlength: especifica o número máximo de caracteres permitidos em um campo de entrada. Entretanto, o mesmo não alerta ao usuário sobre o limite, ficando a cargo do programador.
  • min e max: Especificam os valores mínimos e máximos para um campo de entrada. Funciona em (input numbers e dates)
  • multiple: Especifica que o usuário tem a permissão para inserir mais de um valor em um campo de entrada. Funcona em (input email e file)
  • pattern: Especifica uma expressão regular que verifica o formato do campo de entrada quando ele é enviado. Funciona em ( input text, date, search, url, tel, email e password). Use o atributo global title para dizer ao usuário qual formato está esperando.
  • placeholder: Especifica uma breve dica do valor esperado de um campo de entrada, onde a dica é exibida dentro do próprio campo de entrada, antes do usuário inserir um valor.
  • required: Especifica que um campo de enterada é obrigatório e deve ser preenchido.
  • step: Especifica o intervalo de múltiplos aceitos de um número. Ex. step=2, então serão aceitos somente os valores, ..., -4, -2, 0, -2, -4, -6, etc. Funciona em (input number, range e todas as datas)
  • autofocus: Diz que o campo de entrada deve ter foco assim que a pagina é carregada.
  • list: Se refere a um atributo do elemento <datalist>, que carrega opções pré-definidas (recomendadas) para um elemento <input>.
  • autocomplete: Especifica que um formulário ou campo de entrada deve ter preenchimento automático ativado para que os navegadores, com base no histórico de preenchimento, recomende opções para o usuário.
  • checked: Especifica qual campo de entrada, do tipo radio ou checkbox, deve ter um valor pré-selecionado ao carregar a pagina.

</form>

</form>

input de datas

Existem muitos elementos que trabalham com diferentes formatos de data e horários.

  • <input type="date">: É usado para campos de entrada que devem conter uma data
  • <input type="datetime-local">: É usado para para campos de entrada que devem conter data e hora, sem fuso horário.
  • <input type="month">: É usado para que o usuário escolha um mês e um ano.
  • <input type="time">: É usado para que o usuário escolha um horário sem fuso horário.
  • <input type="week">: É usado para que o usuário escolha uma semana e um ano.
Código Resultado
<form>
<label for="data">aniversário</label><br>
<input id="data" type="date" name="data">
<label for="datahora">Data e hora do nascimento </label><br>
<input id="datahora" type="datetime-local" name="datahora">
<label for="mesano">Mes e ano do nascimento </label><br>
<input id="mesano" type="month" name="mesano">
<label for="hora">Horário</label><br>
<input id="hora" type="time" name="hora">
<label for="semanaano"> A semana e ano do nascimento </label><br>
<input id="semanaano" type="month" name="semanaano">
</form>

Datalist

O elemento <datalist> especifica uma lista de opções predefinidas para um <input>, onde os usuários verão uma lista suspensa das opções prédefinidas (recomendações/<option>) à medida em que inserem os dados.

Código Resultado
<form>
<label for="navegadores">Qual navegador você usa? </label>
<input id="navegadores" list="nav">
<datalist id="nav">
<option value="Internet Explorer">
<option value="Firefox">
<option value="Chrome">
<option value="Opera">
<option value="Safari">
</datalist>
</form>
