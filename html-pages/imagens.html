IMAGENS

Para inserir uma imagem usamos a tag <img> junto com o atributo src="caminho ou link". Além disso, é um elemento vazio, que não precisa de um tag final.

Código Resultado
<img src="imagens/img.png" width="100%">

Os principais atributos relacionados a tag <img> são:

  • alt: Este atributo define um texto alternativo que descreve a imagem. Os Usuários irão ver o texto se a URL da imagem estiver errado, a imagem não está em um dos formatos suportados ou até a imagem ser baixada. Além disso, é um recurso importante para que, através de plugins, a descrição da imagem seja narrada, permitindo uma maior acessibilidade.
  • height: Este atributo define a altura da imagem em HTML5 pixels ou porcentagem.
  • width: Este atributo define a largura da imagem em pixels ou porcentagem.
  • srcset: Este atributo define, em uma lista de uma ou mais strings, separadas por vírgulas, um conjunto de imagens possíveis, onde o navegador escolhe a imagem mais adequada para exibir em um determinado momento.
Código Resultado
<img src="imagens/img.png" style="width:100%; height:200" alt="logo projetando um projeto.">

Lembrando que o HTML deve ser apenas estrutural, sendo assim, é recomendado que o estilo das imagens, bem como a altura e a largura, sejam aplicadas usando o CSS através so atributo style. Como veremos em CSS.



figure e figcaption

O elemento <figure> especifica um conteúdo independente, como fotos, ilustrações, diagramas, códigos, etc. Enquanto o elemento <figcaption> define uma legenda para o elemento <figure>.

O seu uso está mais relacionado com a semântica dos elementos dentro dela, dando um significo maior que apenas uma imagem, por exemplo.

Código Resultado
<figure>
<img src="imagens/img.png" alt="Logo" style="width:100%; height:200">
<figcaption>Fig1. Logotipo da Empresa</figcaption>
</figure>


Mapa de imagens

Um mapa de imagens, é basicamente, uma imagem com áreas clicáveis, que executam uma ação. A tag <map> define um mapa de imagem, em conjunto com a tag <area>.

Primeiro passo, precisamos definir uma imagem, adicionando um atributo chamado usemap, especificando o valor com #nomedomapa.

<img src="imagens/bing-google-yahoo.jpg" alt="sites" usemap="#workmap">

Após definIda a imagem, precismos criar o mapa com mesmo nome do valor especificado.

<map name="workmap"> </map>

Como último e mais complexo passo, precisamos definir as áreas de uma imagem que permite interações. O primeiro atributo define a forma (shape) da aréa, podendo ser retangular, circular, poligonal ou default. O segundo atributo são as coordenadas (coords). Esse atributo tem como valor, pontos da imagens imagem original e/ou largura de um ponto. Além disso, temos o terceiro atributo href que define qual página queremos abrir caso haja interação com aquela área.

<img src="imagens/bing-google-yahoo.jpg" alt="sites" usemap="#workmap">
<map name="workmap">
<area shape="rect" coords="34,44,270,350" alt="Google" href="https://www.google.com">
<area shape="rect" coords="290,172,333,250" alt="Bing" href="https://www.bing.com">
<area shape="circle" coords="337,300,44" alt="Yahoo" href="https://www.yahoo.com">
</map>







Elementos da Imagens

Elementos de imagens são usados para especificar um conjunto de imagens, que de acordo com o tamanho da tela, serão carregadas. As tags envolvidas são o <picture>, que contém uma ou mais tags <source> que especificam través de atributos o tamanho da tela e o arquivo a ser carregado, além da tag <img> que exibirá a imagem caso o navegador não suporte os elementos.

Experimente diminuir e aumentar a tela:

<picture>
<source media="(min-width: 650px)" srcset="img.jpg">
<source media="(min-width: 465px)" srcset="img2.png">
<img style="width: 100%; height: 200px;" src="img.jpg">
</picture>

O atributo media especifica através do valor min-width, o tamanho mínimo que a tela deve ter para exibir a imagem.

Além de aplicar suporte para imagens de diversos tamanhos de telas, evitando que imagens muito grande sejam carregadas para dispositivos limitados, também é possível apenas especificar imagens de diferentes formatos, caso o dispositivo não suporte determinado formato.
