Classes em JavaScript

Uma classes é um modelo para objetos... Ou seja, é uma forma de podermos criar vários objetos com as mesmas funções e propriedades sem termos que criar todo o corpo do objeto múltiplas vezes. Para criarmos uma classe devemos usar a palavra reservada class e devemos começar o nome de uma classe sempre com a primeira letra maiúscula para diferenciá-las de variáveis normais.

Um objeto criado em objetos difere um pouco de sua sintaxe de uma classe pois não precisa ser instanciado, ou dito de outra forma, construído. Para isso, uma classe sempre deve ter um método chamado constructor() { ... } no seu corpo onde irá inicializar as principais propriedades do objeto.

A palavra reservada this cria ou usar uma propriedade dentro da classe com o nome especificado após o ponto.

class Pessoa {
constructor(nome, idade) {
this.nome = nome;
this.idade = idade;
}
}

Construindo objetos

Parar criar então um objeto usando um modelo, basta usar a palavra reservada new seguida pelo nome do modelo e seus parâmetros (que são os usados no construtor).

var pessoa1 = new Pessoa("Fulano de Tal", 60);
var pessoa2 = new Pessoa("Sicrano Beltrano", 25);

Constructor

O método constructor(){ ... } deve sempre ter exatamente este nome, será sempre executado na criação de um novo objeto com seus parâmetros especificados que serão usados para inicializar os valores das propriedades do objeto. Por padrão, caso o método esteja omitido, o JS irá criar um método construtor sem código e sem parâmetros (construtor vazio).

Criando métodos

Para criar métodos em uma classe, basta seguir os mesmo métodos de criação de métodos de um objeto e usá-los da após a construção de um objeto da mesma forma.

Mostrar Código

<div id="classe"></div>
<script>
class Pessoa {
constructor(nome, idade) {
this.nome = nome;
this.idade = idade;
}
falarNome() {
return "Meu nome é : " + this.nome;
}
}
var pessoa = new Pessoa( "Fulano de Tal" , 70);
document.getElementById("classe").innerHTML = pessoa.falarNome();
</script>

Extends - Herança entre classes

Herança é um conceito de Orientação a Objetos, onde uma classe herda (extends) as propriedades e os métodos de uma classe pai. Em caso de heranças múltiplas, o filho do filho, também herdará as propriedades e métodos do pai e do avô.

O método super() é usado para acessar as propriedades e métodos da classe pái. Se o método super() for chamado dentro do método constructor(), ele invocará o método construtor da classe pai para poder inicializar as propriedades do pai.

Podemos também usar um outro conceito de Orientação a Objetos que é o polimorfismo, onde podemos sobrescrever os métodos da classe pai, criando um método de mesmo nome, porém com funcionalidades diferentes.

Mostrar Código

<div id="heranca"></div>
<script>
class Animal {
constructor(especie) {
this.especie = especie;
}
emiteSom() {
return "SOM";
}
viver() {
return "Este Animal está vivo";
}
}

class Cachorro extends Animal {
constructor(nome, idade) {
super("mamífero");
this.nome = nome;
this.idade = idade;
}
emiteSom() {
return "LATIR!";
}
}

class Periquito extends Animal {
constructor(nome, idade) {
super("ave");
this.nome = nome;
this.idade = idade;
}
emiteSom() {
return "PIAR?!";
}
}
var dog = new Cachorro( "Doguinho" , 7);
var bird = new Periquito( "Quiquito" , 1);
document.getElementById("heranca").innerHTML =
"O periquito é uma " + bird.especie +
" e ele emite o som de : " + bird.emiteSom() +
" ----- Então: " + bird.viver() +
"<br> O cachorro é um " + dog.especie +
" e ele emite o som de : " + dog.emiteSom() +
" ----- Então: " + dog.viver();
</script>


Como você viu acima vários conceitos, onde mesmo que bird e dog não tenham em suas classes a propriedade especie e o método viver() eles herdaram estes da classe pai Animal. E mesmo que o Animal tivesse o método emitirSom() usamos o conceito de polimorfismo e sobrescrevemos o método para funcionar de acordo com cada espécie. Por dim temos o detalhe da funcionalidade super() que atribuí o nome da espécie para a classe herdada.

Getters e Setters com classes

Também podemos usar o conceito de get e set com as classes para "proteger" as propriedades de atribuições de valores inválidos, assim como tratar a exibição dos dados. Como visto em Objetos. Como get e setters não podem ter o mesmo nome que as propriedades é um padrão começar os nomes das propriedades com um underline _.

Assim os métodos se comportarão como as propriedades, funcionando de forma totalmente transparente, sem notarmos que estamos usando métodos e não variáveis.

Mostrar Código

<div id="get-set"></div>
<script>
class Pessoa {
constructor(nome, idade) {
this._nome = nome;
this._idade = idade;
}
falarNome() {
return "Meu nome é : " + this._nome;
}
get nome() {
return "Meu nome é : " + this._nome;
}
set nome( nome ) {
this._nome = nome.toUpperCase();
}
get idade() {
return "Meu idade é : " + this._idade;
}
set idade( idade ) {
this._idade = idade;
}
get altura() {
return "Meu altura é : " + this._altura;
}
set altura( Nome ) {
this._altura = altura;
}
}
var pessoa = new Pessoa("Fulanoadasda", 47);
pessoa.nome = "Fulano de Tal Sicrano";
document.getElementById("get-set").innerHTML = pessoa.nome;
</script>

Métodos estáticos

Os métodos estáticos são criados com a palavra reservada static e só podem ser usados ór uma classe, ou seja, se você construir um objeto e tentar usar o método o mesmo retornará erros. única forma de usar um método estático é usando o método dentro da própria classe ou chamando a classe.

Mostrar Código

<div id="estatico"></div>
<script>
class Pessoa {
constructor(nome, idade) {
this.nome = nome;
this.idade = idade;
}
static falarNome() {
return "Meu nome é estático";
}
}
document.getElementById("estatico").innerHTML = Pessoa.falarNome();
</script>
