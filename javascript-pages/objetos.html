Objetos em JavaScript

De forma bem sucinta, um objeto em linguagem de programação é um forma de representar as características de algo real em uma formato computacional. Um exemplo de OBJETO do mundo real, seria um CARRO, onde o carto tem PROPRIEDADES como cor, marca, preço. Além disso, um carro tem certas funcionalidades (MÉTODOS) como frear, acelerar, ligar ou desligar.

  • this - This é usado para dizer que aquela variável pertence ao objeto criado e não uma outra variável de mesmo nome, que pode ser usada como parâmetro de uma função ou variável global.
  • new - New é usado para criar um novo objeto.

Um objeto em JS nada mais é que um variável com funções e múltiplas outras variáveis dentro dela. Vamos criar um objeto computacional do objeto real pessoa. Como observações, cada variável dentro de um objeto deve ser separada por VÍRGULAS sem o uso da palavra reservada var.

var pessoa = {
nome: "Fulano Sicrano de Tal",
idade: 47,
altura: 1.68,
falarNome : function() {
return "Meu nome é : " + this.nome;
}
};

Para acessar uma variável dentro de um objeto devemos usar a sintaxe objeto.variável se for um método deve ser usado objeto.função()

Mostrar Código

<div id="objeto"></div>
<script>
var pessoa = {
nome: "Fulano Sicrano de Tal",
idade: 47,
altura: 1.68,
falarNome : function() {
return "Meu nome é : " + this.nome;
}
};
document.getElementById("objeto").innerHTML = pessoa.falarNome();
</script>

Exibindo o objeto

Podemos exibir o corpo de um objeto devemos usar o método Object.values(objeto) para retornar um array com todos os valores dentro de um objeto.

Mostrar Código

Getters e Setters

As palavras reservadas get e set são usada para "proteger" o acesso das propriedades dentro de um objeto, onde para pegar o valor de uma propriedade devemos usar os métodos get, enquanto que para alterar os valores das propriedades de um objeto devemos usar os métodos set.

Os métodos set get não podem ter o mesmo nome que as propriedades, existe um padrão de uso que iremos falar em Classes, mas, por enquanto, não iremos detalhar. E mesmo que esses métodos sejam um método não é necessário usar () para invocar estes métodos.

Mostrar Código

<div id="get-set"></div>
<script>
var pessoa = {
nome: "Fulano Sicrano de Tal",
idade: 47,
altura: 1.68,
get getNome() {
return "Meu nome é : " + this.nome;
}
set setNome( nome ) {
this.nome = nome.toUpperCase();
}
get getIdade() {
return "Meu idade é : " + this.idade;
}
set setIdade( idade ) {
this.idade = idade;
}
get getAltura() {
return "Meu altura é : " + this.altura;
}
set setAltura( Nome ) {
this.altura = altura;
}
};
pessoa.setNome = "Fulano de Tal Sicrano"
document.getElementById("get-set").innerHTML = pessoa.getNome;
</script>


Assim podemos colocar dentro da função set nome uma função para proteger o dado que entra e polir o dado que sai pelos métodos get. Claro que as propriedades originais ainda poderão ser acessadas pois o JS não possui modificadores de acessos, mas fica como um contrato em quem for usar e quer disponibilizar que não irá acessar as demais propriedades.

Objetos como Funções

Funções construtores são nomeadas com a primeira letra em maiúsculo. A diferença é que as funções construtores não são objetos, mas podem criar um ou mais objetos, e não podem conter métodos dentro delas de imediato. Para construir um objeto precisamos usar a palavra reservada new seguida pelo nome da função e seus parâmetros.

Mostrar Código

<div id="functions"></div>
<script>
function Pessoa (nome, idade, altura) {
this.nome = nome;
this.idade = idade;
this.altura = altura;
this.falarNome = function () {
return "Meu nome é : " + this.nome;
}
};
var alguem = new Pesssoa("fulano", 37, 1.75);
document.getElementById("functions").innerHTML = alguem.falarNome();
</script>
