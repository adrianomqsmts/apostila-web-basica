Ajax

O JAX (Asynchronous JavaScript And XML) não é uma linguagem de programação, mas sim uma técnica de acessar o o servidor web de uma determinada página web. A Base do AJAX consiste no objeto interno do navegador chamado XMLHttpRequest que é responsável para solicitar os dados do servidor.

A principal vantagem do AJAX é que ele nos permite trabalhar com páginas assíncronas, ou seja, a troca de mensagem com o servidor ocorre por trás sem o o usuário saiba dessa interação, e a página WEB fica parecendo uma aplicação desktop, pois com esta técnica poderemos atualizar trechos de uma página sem termos que recarregar toda a página.

O clico de vida do ajax consiste em:

  1. Ocorre um evento em uma página web
  2. Um objeto XMLHttpRequest é criado por JavaScript
  3. O objeto XMLHttpRequest envia uma solicitação a um servidor web
  4. O servidor processa o pedido
  5. O servidor envia uma resposta de volta à página da web
  6. A resposta é lida pelo JavaScript

Escrever código AJAX regular pode ser um pouco complicado, porque navegadores diferentes têm sintaxes diferentes para implementação de AJAX. Isso significa que você terá que escrever código extra para testar diferentes navegadores. No entanto, a equipe jQuery cuidou disso para nós, para que possamos escrever a funcionalidade AJAX com apenas uma única linha de código.

load()

O método load() jQuery é um método AJAX simples, mas poderoso. O método carrega dados de um servidor e coloca os dados retornados no elemento selecionado.

Sinaxe: $(selector).load(URL,data,callback);

  • O parâmetro de URL necessário especifica a URL que deseja carregar.
  • O parâmetro de dados opcional especifica um conjunto de pares de chave/valor de consulta para enviar junto com a solicitação.
  • O parâmetro opcional de retorno de chamada é o nome de uma função a ser executada após a conclusão do método.

O exemplo abaixo irá carregar todo o conteúdo HTML dentro do elemento com ID chamado div, após o click sobre o elemento com ID btn.

$(document).ready(() => {
  $('#btn').on('click', () => {
    $('#div').load('exemplo-ajax.html');
  })
});

Também é possível adicionar um seletor jQuery ao parâmetro URL. O exemplo a seguir carrega o conteúdo do elemento com id="p1", dentro do arquivo "exemplo-ajax.html", em um elemento específico:

$(document).ready(() => {
  $('#btn').on('click', () => {
    $('#div').load('exemplo-ajax.html #p1');
  })
});

O parâmetro opcional de retorno de chamada especifica uma função de retorno de chamada para ser executada quando o método estiver concluído. A função de retorno de chamada pode ter parâmetros diferentes:

  • responseTxt - contém o conteúdo resultante se a chamada for bem sucedida
  • statusTxt - contém o status da chamada
  • xhr - contém o objeto XMLHttpRequest

O exemplo a seguir exibe uma caixa de alerta após o método de carga() ser concluído. Se o método tiver sido bem sucedido, ele exibirá "Conteúdo externo carregado com sucesso!", e se falhar, ele exibirá uma mensagem de erro:

$("button").click(function(){
  $("#div1").load("demo_test.txt", function(responseTxt, statusTxt, xhr){
    if(statusTxt == "success")
      alert("External content loaded successfully!");
    if(statusTxt == "error")
      alert("Error: " + xhr.status + ": " + xhr.statusText);
  });
});

get() e post()

Os métodos jQuery get() e post() são usados para solicitar dados do servidor com uma solicitação HTTP GET ou POST.

  • GET - Solicita dados de um recurso especificado
    • GET é basicamente usado para apenas obter (recuperar) alguns dados do servidor. Nota: O método GET pode retornar dados armazenados em cache.
  • POST - Envia dados para serem processados em um recurso especificado
    • O POST também pode ser usado para obter alguns dados do servidor. No entanto, o método POST NUNCA armazena dados e é frequentemente usado para enviar dados junto com a solicitação.

O método GET tem sintaxe: $.get(URL,callback);. O parâmetro de URL necessário especifica a URL que deseja solicitar. O parâmetro opcional de retorno de chamada é o nome de uma função a ser executada se a solicitação for bem sucedida. Além disto o segundo parâmetro é uma função de retorno de chamada. O primeiro parâmetro de retorno de chamada contém o conteúdo da página solicitada e o segundo parâmetro de retorno de chamada mantém o status da solicitação. O exemplo a seguir usa o método para recuperar dados de um arquivo no servidor:

$("button").click(function(){
  $.get("demo_test.asp", function(data, status){
    alert("Data: " + data + "\nStatus: " + status);
  });
});

O método POST solicita dados do servidor usando uma solicitação HTTP POST, e tem como sintaxe $.post(URL,data,callback);. O parâmetro de URL necessário especifica a URL que deseja solicitar. O parâmetro de dados opcional especifica alguns dados para enviar junto com a solicitação, o terceiro parâmetro é uma função de retorno de chamada. O primeiro parâmetro de retorno de chamada contém o conteúdo da página solicitada e o segundo parâmetro de retorno de chamada mantém o status da solicitação.. O parâmetro opcional de retorno de chamada é o nome de uma função a ser executada se a solicitação for bem sucedida. O exemplo a seguir usa o método para enviar alguns dados junto com a solicitação:

$("button").click(function(){
  $.post("demo_test_post.asp",
  {
    name: "Donald Duck",
    city: "Duckburg"
  },
  function(data, status){
    alert("Data: " + data + "\nStatus: " + status);
  });
});

Ajax()

Todos os métodos jQuery AJAX usam o método ajax(). Este método é usado principalmente para solicitações onde os outros métodos não podem ser usados. Alguns dos principais parâmetros são:

  • type - Especifica o tipo de solicitação. (GET ou POST)
  • url - Especifica o URL para enviar a solicitação. O padrão é a página atual
  • data - Especifica os dados a serem enviados para o servidor
  • dataType - O tipo de dados esperado da resposta do servidor.
  • erro (xhr, status, erro) - Uma função a ser executada se a solicitação falhar.
  • timeout - O tempo limite local (em milissegundos) para a solicitação

O exemplo abaixo envia uma solicitação para o arquivo "app.php" com o dado selecionado em um elemento SELECT que sofreu um evento de mudança (ou seja, o cliente selecionou outro valor do select). Em seguida, o arquivo PHP retorna a resposta (Response) em formato esperado de JSON, onde, em caso de sucesso alteramops os dados no HTML, caso contrário exibimos o erro no console do navegador.

<script>
  $(document).ready(() => {
    //ajax
    $('#select').on('change', e => {
      // parametros -> método, url, dados, sucesso e erro

      let valor = $(e.target).val();

      $.ajax({
        type: 'GET',
        url: 'app.php',
        data: `valor=${valor}`,
        dataType: 'json',
        success: dados => {
          $('#totalVendas').html(dados.totalVendas);
          $('#numeroVendas').html(dados.numeroVendas);
        },
        error: erro => { console.log(erro) }
      })
   })
  })
</script>
