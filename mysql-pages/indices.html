Criando índices com MySQL

Em banco de dados completos, ficar realizando constantes pesquisar podem acarretar em uma perda de desempenho, se toda a vez o sistema tive que buscar dos dados e ficar comparando as colunas. E em uma forma de otimizar, principalmente as pesquisas, surgiu a instrução CREATE INDEX para agilizar a pesquisa, evitando que o MySQL faça pesquisas olhando em todas as colunas.

CREATE TABLE tabela_do_brasileirao(
  pos_time INT,
  nm_time VARCHAR(45),
  est_time CHAR(2),
  qtde_vit INT,
  qtde_emp INT,
  qtde_der INT,
  gols_feitos INT,
  gols_sofridos INT,
  qtde_pontos INT
);

CREATE INDEX - Criando índices

O comando CREATE INDEX é usado para criar index que otimizam as pesquisas em um banco de dados. A sua sintaxe base é CREATE INDEX nome_index ON tabela (coluna1, coluna2,..);. Em seguida para usar um índice em uma pesquisa, basta usar a instrução USE INDEX nome_index. A segunda forma de criar um índice é usando a instrução ALTER TABLE tabela ADD INDEX nome_index (coluna1, coluna2, ...);.

CREATE INDEX times_index ON tabela_do_brasileirao (nm_time);
-- OU
ALTER TABLE tabela_do_brasileirao ADD INDEX golsf_index (gols_feitos);

-- Usando o index para pesquisas
SELECT nm_time, est_time, qtde_vit, gols_feitos, qtde_pontos FROM tabela_do_brasileirao USE INDEX (brasil_index) WHERE nm_time = 'GOIÁS';

Podemos também usar múltiplas colunas em um index

CREATE INDEX index_triplo ON tabela_do_brasileirao (qtde_vit, qtde_emp, qtde_der);

SELECT nm_time, est_time, qtde_vit, gols_feitos, qtde_pontos from tabela_do_brasileirao USE INDEX (index_triplo) WHERE NM_TIME = 'ATLÉTICO';

DROP INDEX - Removendo os índice

A instrução ALTER TABLE tabela DROP nome_index é usado para remover o índice de uma tabela.

ALTER TABLE tabela_do_brasileirao DROP INDEX INDEX_TRIPLO;

ALTER TABLE tabela_do_brasileirao DROP INDEX GOLS_INDEX;
