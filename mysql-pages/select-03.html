Selecionando múltiplas tabelas MySQL

Para relacionarmos duas tabelas e seus dados sejam correlacionados de forma correta, sem duplicação de dados, precismos trabalhar com as chaves primárias e as chaves estrangeiras. Basicamente devemos usar a instrução SELECT e usar a cláusura WHERE com a condição de igualdade entre a chave primária de uma tabela com a chave estrangeira de outra tabela.

O que pode correr é de uma coluna na tabela A ter o mesmo nome que uma coluna na tabela B, e para referenciar que aquela coluna pertence a uma tabela específica devemos usar a sintaxe nomeTable.coluna, uma forma mais sucinta é dar um apelido para a nome da tabela na consulta com a instrução nomeTable AS n resultando em um sintaxe mais curta como n.coluna. Este exemplo pode ser notado nos exemplos abaixo.

CREATE TABLE produtos(
  -- Chave primária do produto
  id_produto INT primary key auto_increment,
  nome TEXT,
  preco DOUBLE
);

CREATE TABLE pedidos(
  -- Chave primária do pedido
  id_pedido INT primary key,
  id_produto INT,
  quantidade INT,
  -- Criando uma chave estrageira do produto no pedido
  foreign key (id_produto) references produtos (id_produto)
);

-- Selecionando duas tabelas apelidadas e lingando os dados entre elas através das chaves id_produto
SELECT ped.id_pedido, pro.id_produto, pro.nome, pro.preco, ped.quantidade 
FROM produtos AS pro, pedidos AS ped

JOIN - Selecionando múltiplas tabelas

A cláusula JOIN é um sintaxe alternativa para a cláusula WHERE, onde ele irá juntar duas tabelas, de forma correta, sem espelhamento de dados dada uma condição. A sua sintaxe base é SELECT coluna1, coluna2,... FROM tabelaA JOIN tabelaB ON condição;. A condição mais comum é a igualdade dentre a chave primária da tabela A com a chave estrangeira da tabela B.

-- Selecionando duas tabelas apelidadas e lingando os dados entre elas através das chaves id_produto
SELECT ped.id_pedido, pro.id_produto, pro.nome, ped.quantidade, (pro.preco*ped.quantidade) AS total
FROM produtos AS pro JOIN pedidos AS ped ON ped.id_produto = pro.id_produto;

INNER JOIN

A cláusula INNER JOIN é parecido com a cláusula JOIN ON anterior, entretanto ela irá forçar com que o resultado seja igual na condição ON. A instrução completa retornará a intersecção entre as duas tabelas de acordo com a condição. A sua sintaxe é SELECT coluna1, coluna2,... FROM tabelaA INNER JOIN tabelaB ON condição;.


CREATE TABLE clientes(
  cod_cli INT primary key,
  nome_cli VARCHAR(45),
  rua_cli VARCHAR(45),
  bairro_cli INT,
  num_cli INT,
  fone_cli VARCHAR(9)
);

CREATE TABLE carros(
  placa_car VARCHAR6(45)) primary key,
  cliente_car INT,
  marca_car VARCHAR(45),
  modelo_car VARCHAR(45),
  cor_car VARCHAR(45)
);

ALTER TABLE carros add foreign key (cliente_car) references clientes (cod_cli);

CROSS JOIN

A cláusula CROSS JOIN é usada para espelhar os dados, ou seja, o primeiro dado da tabela irá ser combinado com todos os dados da segunda tabela, então o segundo dado da primeira tabela também será combinado com todos os dados da segunda tabela, e assim sucessivamente. A sintaxe base é SELECT coluna1, coluna2,... FROM tabelaA CROSS JOIN tabelaB. Se usada com a condição ON o resultado será igual a cláusula INNER JOIN.

SELECT nome_cli, carros.cliente_car, carros.placa_car FROM clientes CROSS JOIN carros;

RIGHT (outer) JOIN

A cláusula RIGHT JOIN é usada para mostrar tudo da tabela da direita em relação a tabela da esquerda. Basicamente, ele irá mostrar todos os dados da tabela da direita combinados com os dados da tabela da esquerda. Onde não houver a combinação, então os dados das colunas provenientes da tabela da esquerda serão nulos.. A sintaxe base é SELECT coluna1, coluna2,... FROM tabelaA RIGHT OUTER JOIN tabelaB ON condição (o OUTER é opcional).


-- Iremos selecionar todos os carros que possuem ou não clientes
SELECT clientes.nome_cli, carros.cliente_car, carros.placa_car
FROM clientes RIGHT JOIN carros ON cod_cli = cliente_car;

LEFT (outer) JOIN

A cláusula LEFT JOIN é igual a consulta acima, entretanto, ela dá prioridade para a tabela da esquerda. Basicamente, ele irá mostrar todos os dados da tabela da esquerda combinados com os dados da tabela da direita. Onde não houver a combinação, então os dados das colunas provenientes da tabela da direita serão nulos. SELECT coluna1, coluna2,... FROM tabelaA LEFT OUTER JOIN tabelaB ON condição (o OUTER é opcional).


-- Iremos selecionar todos os clientes que possuem ou não carros
SELECT clientes.nome_cli, carros.cliente_car, carros.placa_car
FROM clientes LEFT JOIN carros ON cod_cli = cliente_car;

FULL JOIN

A cláusula FULL JOIN é usada para selecionar tanto os dados da tabela da direita quanto a tabela da esquerda. Entretanto, o MySQL não tem suporte para a cláusula. Para simular o FULL JOIN precisamos fazer a união UNION entre a consulta com a cláusula LEFT JOIN e a cláusula RIGHT JOIN.

SELECT clientes.nome_cli, carros.cliente_car, carros.placa_car
FROM clientes LEFT JOIN carros ON cod_cli = cliente_car
UNION
SELECT clientes.nome_cli, carros.cliente_car, carros.placa_car
FROM clientes RIGHT JOIN carros ON cod_cli = cliente_car;
