Tipos de dados em MySQL

Em SQL precisamos informar o tipo de dado para cada coluna de uma tabela, podendo até formatar este dado. Abaixo temos ums lista completa com todos os dados aceitos e uma breve descrição de como funcionam.

Dados do tipo String


Tipo de Dado Descrição
CHAR(tamanho) Usado para strings de tamanho fixo (como cpf, telefone, etc). O tamanho especificado pode variar entre 0 até 255 caracteres, e caso não seja especificado, o tamanho padrão é 1
VARCHAR(tamanho) Usado para string de tamanho variáveis. O tamanho especificado pode variar entre 0 até 65535 caracteres
BINARY(size) Igual ao CHAR(), mas é usado para armazenar string binárias.
VARBINARY(size) Igual ao VARCHAR(), mas é usado para armazenar string binárias.
TINYBLOB Usado para BLOBs (Binary Large OBjects). Tamanho máximo de 255 bytes
TINYTEXT Contém uma string com comprimento máximo de 255 caracteres
TEXT(size) Contém uma string com comprimento máximo de 65.535 bytes
BLOB(size) Para BLOBs (Binary Large OBjects). Armazena até 65.535 bytes de dados
MEDIUMTEXT Contém uma string com comprimento máximo de 16.777.215 caracteres
MEDIUMBLOB Para BLOBs (Binary Large OBjects). Armazena até 16,777,215 bytes de dados
LONGTEXT Contém uma string com comprimento máximo de 4.294.967.295 caracteres
LONGBLOB Para BLOBs (Binary Large OBjects). Armazena até 4,294,967,295 bytes de dados
ENUM(val1, val2, val3, ...) Um tipo string que somente poderá ter o valor especificado dentro do parâmetro. Um enum() suporta até 65535 valores em uma lista. Caso hja a inserção de um valor não especificado na lista, então o valor inserido será em branco.
SET(val1, val2, val3, ...) Um objeto string que pode ter 0 ou mais valores, escolhidos em uma lista de valores possíveis. que podem ter até 64 valores.

Tipo de dados numéricos


UNSIGNED é usado para especificar valores inteiros somente

Tipo de Dado Descrição
BIT() Usado para armazenar valores de bit. Onde o tamanho do valor armazenado varia de 1 até 64, sendo 1 o valor padrão.
TINYINT Usado para armazenar um valor de inteiro muito pequeno. Os valores suportados variam de -128 até 127. Valores positivos variam de 0 até 255 (128 + 127 = 255)
BOOL ou BOOLEAN Usado para armazenar valores booleanos, onde 0 é falso (false), enquanto qualquer outro valor é verdadeiro (true).
SMALLINT Usado para armazenar um valor de inteiro pequeno. Os valores suportados variam de -32768 até 32767. Valores positivos variam de 0 até 65535
MEDIUMINT Usado para armazenar um valor de inteiro quase pequeno. Os valores suportados variam de -8388608 até 8388607. Valores positivos variam de 0 até 16777215
INT ou INTEGER Usado para armazenar um valor de inteiro. Os valores suportados variam de -2147483648 até 2147483647. Valores positivos variam de 0 até 4294967295
BIGINT Usado para armazenar um valor de inteiro grande. Os valores suportados variam de -9223372036854775808 até 9223372036854775807. Valores positivos variam de 0 até 4294967295 18446744073709551615
FLOAT(p) ou FLOAT Usado para armazenar valores decimais. O valor de p determina se o valor exibido deve ser ponto flutuante (0 a 23) ou precisão dupla DOBLE (24 a 53)
DOUBLE Usado para armazenar valores decimais com precisão dupla.
DECIMAL(tamanho, d) ou DEC(tamanho, d)
ou NUMERIC(tamanho, d)		 Usado para armazenar valores decimais, especificando quantidade de números aceitos, e o parâmetro d especifica quantas casas decimais deve ter o valor.

Tipo de dados para Datas


Tipo de Dado Descrição
DATE Usado para armazenar uma data no formato YYYY-MM-DD.
DATETIME Usado para armazenar uma data e hora no formato YYYY-MM-DD hh:mm:ss. Para inserir a data atual pode usar DEFAULT CURRENT_TIMESTAMP ou ON UPDATE CURRENT_TIMESTAMP
TIMESTAMP Usado para inserir uma data e hora UNIX no formato YYYY-MM-DD hh:mm:ss. Para inserir a data atual pode usar DEFAULT CURRENT_TIMESTAMP ou ON UPDATE CURRENT_TIMESTAMP
TIME Usado para inserir um hora no formato hh:mm:ss. Os calores suportados são '-838:59:59' até '838:59:59'
YEAR Usado para inserir o ano no formato de 4 dígitos. Os valores suportados são de 1901 até 2155, além do ano 0000.
