Usuário do SGBD MySQL

O MySQL nos permite criar usuários para um banco de dados e dar terminadas permissões para o usuário, como apenas ler o conteúdo de uma tabela, apenas inserir dados em outra determinada tabela, e por assim em diante. Este conteúdo visa a segurança e integridade dos dados evitando acesso indevido.

CREATE USER - Criando usuários

A instrução CREATE USER é usada para criar um usuário e não é necessário estar com um banco de dados ativo. A sintaxe base é CREATE USER 'usuário@localhost' IDENTIFIED BY 'senha';. Em seguida é importante atualizar o sistema para que ele reconheça os novos usuários através do comando FLUSH PRIVILEGES;. Outro detalhe é que o nome do usuário leva em conta minúsculo e maiúsculo.

Para verificar os usuários cadastrados no SGBD, primeiro precisamos ativar o banco de dados do MySQL com a instrução USE MYSQL;, e então usamos a instrução SELECT * FROM user;. E para ver as permissões usuário logado usamos a instrução SHOW GRANTS;. Por padrão um novo usuário não tem nenhuma permissão.

-- Criano um usuário Gerente
CREATE USER 'GERENTE'@'LOCALHOST' IDENTIFIED BY '123';
-- Criando um usupario Empregado
CREATE USER 'EMPREGADO'@'LOCALHOST' IDENTIFIED BY '321';
-- Ataualizando o sistema para reconhecer os novos usuários
FLUSH PRIVILEGES;

-- Acessando o banco de dados interno do MySQL
USE MYSQL;
-- Vendo a tabela de usuários cadastrado no MySQL
SELECT * FROM user;

GRANT - Dando permissões

A instrução GRANT é usada para dar permissões para os usuários previamente criamos no sistema. A sintaxe base é GRANT lista_de_privilégios ON nome_banco_de_dados.tabela TO 'usuário@localhost'. A lista de privilégios pode ver (select, update, insert ou delete) enquanto o nome da tabela do banco de dados pode ser * (asteríscos) para dar as permissão listadas para todas as tabelas daquele banco de dados para o usuário.


-- Dando acesso total para o gerente
GRANT delete, update, insert, select ON empresa.* TO 'GERENTE'@'LOCALHOST';
flush privileges;

Para dar acesso mais específico, como atualizar apenas uma coluna de uma tabela , podemos especificar a tabela, na lista de privilégios, entre parênteses, devemos colocar as colunas que o usuário tem permissão.


-- Se estivermos com o banco de dados ativo, não precisamos escrever o seu nome
USE empresa;


CREATE TABLE funcionarios(
  id_func INT,
  nome VARCHAR (45),
  email VARCHAR (45),
  salario DOUBLE
);


-- O empregado vai poder apenas atualizar o email na tabela funcionário e ver os dados da tabela
GRANT update(email), select ON funcionarios TO 'EMPREGADO'@'LOCALHOST';
flush privileges;

REVOKE - Removendo permissões

A instrução REVOKE é usada para remover permissões de um usuário. A sua sintaxe base é REVOKE lista_de_privilégios ON nome_banco_de_dados.tabela TO 'usuário@localhost', onde os seus detalhes de implementação são iguais ao GRANT, podendo remover permissões em apenas um coluna de uma tabela se assim desejar.


-- Removendo as permissções de criar e inserir do gerente de todas as tabelas do bacno de dados empresa
REVOKE insert, create ON empresa.* FROM 'GERENTE'@'LOCALHOST';
Flush privileges;



IMPORTANTE! Para que o REVOKE e o GRANT sejam de fato implementados os usuários precisam sair e entrar com seus suários do banco de dados.

DROP USER - Removendo usuários

O comando DROP USER é usado para remover o usuário de um banco de dados. A sua sintaxe base é DROP USER 'usuário'@'localhost';.

DROP USER 'EMPREGADO'@'LOCALHOST';
 
DROP USER 'GERENTE'@'LOCALHOST';
