Orientação a Objetos com PHP

De forma bem sucinta, um objeto em linguagem de programação é um forma de representar as características de algo real em uma formato computacional. Um exemplo de OBJETO do mundo real, seria um CARRO, onde o carto tem PROPRIEDADES como cor, marca, preço. Além disso, um carro tem certas funcionalidades (MÉTODOS) como frear, acelerar, ligar ou desligar.

  • this - This é usado para dizer que aquela variável pertence ao objeto criado e não uma outra variável de mesmo nome, que pode ser usada como parâmetro de uma função ou variável global.
  • new - New é usado para criar/instanciar um novo objeto.

Classes

Uma classe é um modelo para objetos... Ou seja, é uma forma de podermos criar vários objetos com as mesmas funções e propriedades sem termos que criar todo o corpo do objeto múltiplas vezes. Para criarmos uma classe devemos usar a palavra reservada class e devemos começar o nome de uma classe sempre com a primeira letra maiúscula para diferenciá-las de variáveis normais. Uma classe tem portanto as variáveis e funções que definem o contexto do objeto.

Todas as variáveis deve ter um tipo de encapsulamento, entretanto, no momento elas serão públicas, e as definimos com a palavra reservada public. Enquanto o método __construct pode ser omitido, mas é muito usado para inicializar variáveis importantes de uma classe. O método é sempre chamado na criação de um objeto, e muitas vezes usamos a palavra reservada $this seguida por uma "seta" -> para especificar a variável da classe e contextualizá-la. Por padrão, caso o método esteja omitido, o PHP criará um método construtor sem código e sem parâmetros (construtor vazio).

Por fim, o método __destruct é sempre executado quando destruímos uma variável, por exemplo, com a função unset($objeto).

<?php
  class Produto{
    public $preco;
    public $validade;
    public $marca;

   function __construct($preco, $validade, $marca) {
      $this->preco = $preco; 
      $this->validade = $validade;
      $this->marca = $marca;
    }

    function desconto($desconto){  
      $this->preco *= (1+$desconto);  
    }

    function __destruct()    {
      return "Este objeto foi destruído";      
    }

  }
?>

Objeto

Para instanciar um objeto, precisamos criar uma variável, usar o operador se atribuição = seguido pela palavra reservada new e o nome da classe Classe(parâmetros) com os parâmetros usados dentro do método construtor. Para acessarmos um atributo ou um método de um objeto, devemos usar o nome do objeto, seguido pelo o operador -> e o nome da propriedades ou método.

  $chapeu = new Produto(10.00, "11/10/2030", "Fulano S/A" );
  $chapeu->desconto(0.10); //Aplica 10% de desconto do preço do produto
  echo $chapeu->preco; //Exibe 9 pois 10 * (0.9) = 9.

Encapsulamento/Modificadores de acesso

Os modificadores de acesso definem quem tem acesso a uma determinada propriedade ou a um determinado métido. Existem três modificadores de acesso:

  • public - A propriedade ou método pode ser acessado de qualquer lugar. Este é o padrão
  • protected - A propriedade ou método pode ser acessado dentro do classe e por classes derivadas dessa classe
  • private - A propriedade ou método pode só poderá ser acessado dentro da classe
<?php
  class Produto{
    public $preco;
    protected $validade;
    private $marca;

   function __construct($preco, $validade, $marca) {
      $this->preco = $preco; 
      $this->validade = $validade;
      $this->marca = $marca;
    }
    private function desconto($desconto){  
      $this->preco *= (1-$desconto);  
    }

  }

  $chapeu = new Produto(10.00, "11/10/2030", "Fulano S/A" );
  $chapeu->desconto(0.10); // Ocorre um erro, pois desconto só pode ser usado pela classe
  $chapeu->validade; // Ocorre um erro, pois a validade so pode ser usada pela classe ou classes herdadas
  $chapeu->preco; // Pode ser usado
  
?>

Get e Set

Get e Set são usados para proteger o acesso direto à uma propriedade de uma classe, como colocar valores negativos em uma idade, ou formatar como deve ser exibido uma determinada propriedade. Assim, ao combinarmos os modificadores de acesso e o conceito de get e set, manteremos uma consistência e uma segurança maior sobre nossos dados.

<?php
  class Produto  {
    private $preco;
    private $marca;

    function __construct($preco, $marca)  {
      $this->preco = $preco;
      $this->marca = $marca;
    }

    function getPreco()  {
      return "R$" + $this->preco;
    }
    
    function getMarca()  {
      return $this->marca + "S/A";
    }

    function setPreco($preco)  {
      if($preco <=0){
        echo "Valor inválido";
      }else{
        $this->preco = $preco;
      }
    }

    function setMarca($marca)  {
      $this->marca = $marca;
    }
  }
?>

Os método SET podem ser chamados dentro do construtor para impedir que na criação do objeto também haja algum valor inválido.

Outra forma de declarar GET e SET é usando um padrão de método chamado __GET(propriedade) e __SET(propriedade, valor) que irá atribuir um valor a uma determinada propriedade ou retornar um valor dada uma propriedade. Esta convenção só é útil quando todos os métodos SET e GET possuem o mesmo corpo de código, variando apenas o nome das propriedades e valores. Assim, ao invés de usarmos $this->propriedade, iremos usar $this->$propriedade, pois agora a propriedade pode variar e não é fixa.

<?php
  class Produto  {
    private $preco;
    private $marca;

    function __construct($preco, $marca)  {
      $this->preco = $preco;
      $this->marca = $marca;
    }

    function __get($propriedade) {
      return $this->$propriedade;
    } 

    function __set($propriedade, $valor) {
      $this->$propriedade = $valor;
    }
  }

  $chapeu = new Produto(10.99, "Fulano S/A");
  echo $chapeu->__get("preco") . "<br>"; // Retorna 10.99
  $chapeu->__set("marca", "Sicrano S/A");
  echo $chapeu->__get("marca"); // Retorna Sicrano S/A
?>

Extends - Herança entre classes

Herança é um conceito de Orientação a Objetos, onde uma classe herda (extends) as propriedades e os métodos de uma classe pai. Em php não existe heranças múltiplas e este problema pode ser resolvido usando traits que veremos mais a frente.

Caso o método __construct não seja definido no filho, ele irá herdar o construtor do pai!

<?php
  class Empresa{
    protected $nome;
    function __construct($nome)  {
      $this->nome = $nome;
    }
    protected function getMarca(){
      return $this->nome;
    } 

    function exibirSlogan(){
      return "Somos tudo!";
    }
  }

  class Produto extends Empresa{
    public $preco;
    protected $validade;
    public $marca;

   function __construct($preco, $validade, $marca) {
      $this->preco = $preco; 
      $this->validade = $validade;
      //Herdou o nome e o método da classe empresa
      $this->nome = $marca;
      $this->marca = $this->getMarca(); 
    }

  }

  $chapeu = new Produto(10.00, "11/10/2030", "Fulano S/A" );

  echo $chapeu->marca; //Exibe o nome da marca

  // ERRO apesar de herdar a propriedade nome, a propriedade é protected, 
  // Ou seja, só pode ser usada pela classe empresa e produto.
  echo $chapeu->nome;
  
  echo $chapeu->exibirSlogan(); //Exibe o slogan herdado

   // ERRO apesar de herdar o método marca, o método é protected, 
   // Ou seja, só pode ser usada pela classe empresa e produto.
  echo $chapeu->marca();
?>

Sobreposição de métodos

A sobreposição de métodos, ocorre quando sobrescrevemos um método herdado. Basicamente herdamos um método da classe pai, mas criamos no corpo do filho um função com o mesmo nome da classe pai, mas com um corpo de código diferente. Isso irá sobrescrever a função do pai com o mesmo método.

<?php
  class Animal{
    function falar(){
      return "";
    }
  }

  class Cachorro extends Animal{
    function falar(){
      return "au au au!";
    }
  }

  class Gato extends Animal{
    function falar(){
      return "miau!";
    }
  }
?>

Final - impedindo a herança ou a sobrescrita

A palavra reservada final é usada em propriedades, métodos e até classes para impedir a herança ou a sobrecarga. Basicamente uma classe final não pode ser herdada, enquanto métodos finais não podem ser sobrescrito.

<?php
  final class Animal{
  }

  class Ave{
    final function voar(){
      echo "Estou voando!";
    }
  }

// Erro, a classe animal não pode ser herdada
  class Cachorro extends Animal{
  }

  class Papagaio extends Ave{
    //Erro o método voar não pode ser sobrescrito
    function voar(){
      return "Vou voar e falar";
    }
  }

?>

Classes e métodos abstratos

Classes e métodos abstratos são classes e métodos sem um corpo definido, ou seja, sem códigos. Por definição, um classe abstrata deve conter pelo menos um método abstrato e um método abstrato deve conter apenas a declaração do seu nome, sem código em seu corpo. Para criá-los devemos usar a palavra reservada abstract.

O objetivo de uma classe abstrata ou método abstrato é "obrigar" que as classes filhos (que herdem de seus pais) implementem estes métodos e definam o corpo dos métodos. Então, ao herdar um classe abstrata, o método da classe filha deve ser definido com o mesmo nome e o mesmo (ou menos restrito) modificador de acesso do que o método da classe pai.

Por definição métodos abstratos são sempre protected, portanto o método sobrescrito da classe filha deve ser protected ou public e deve ter pelo menos os mesmo parâmetros que a classe pai (mesma quantidade e tipo), ou então mais parâmetros opcionais, ou seja, que já tenham algum valor pré-definido.

<?php
  abstract class Animal  {
    public $nome;
    function __construct($nome)    {
      $this->nome = $nome;
    }

    abstract public function emitirSom(): string;
    abstract public function dormir($horas): int;
    abstract protected function comer($comida);
  }

  class Cachorro extends Animal{

    public function emitirSom(): string  {
      return "Au au au";
    }

    public function dormir($horas): int  {
      return $horas;
    }

    function comer($comida, $prato = "chão")  {
      echo "Comendo " . $comida . " no " . $prato;
    }
  }

  $animal = new Cachorro("doguinho");

  $animal->comer("ração","prato");
  echo "<br>";
  $animal->comer("carne");
  echo "<br>";
  echo $animal->dormir(12) . "<br>";
  echo $animal->emitirSom() . "<br>";

?>

Interfaces

Uma interface tem quase a mesma lógica de uso que uma classe abstrata, entretanto, uma interface não é uma classe, não pode conter propriedades, seus métodos só podem ser públicos e abstratos e uma classe pode implementar múltiplas interfaces. Para criar uma interface devemos usar a palavra reservada interface e para "herdar" uma interface, usamos a palavra reservada implements. Uma interface pode ser vista também como um contrato, onde a classe deve cumprir e ter todos os seus métodos implementados.

<?php
  interface Animal{
    public function emitirSom(): string;
  }

  class Cachorro implements Animal{

    public function emitirSom(): string  {
      return "Au au au";
    }
  
  }

  $animal = new Cachorro();
  echo $animal->emitirSom();
?>

Traits - Burlando a herança única

Traits são usados para declarar múltiplos métodos que podem ser usados em várias classes. Os métodos dentro de um bloco trait podem ser abstratos e até mesmo conter modificadores de acessos. Para usar um trait dentro de uma classe devemos usar a palavra reservada use seguida pelo nome do trait dentro do corpo da classe.

<?php
  trait Animal{
    public function viver(){
      echo "Estou VIVO!";
    }
    abstract public function comer($comida) : string;
  }

  class Cachorro {
    use Animal;
    public function comer($comida) : string{
      return "Estou comendo" . $comida;
    }
  }

  $animal = new Cachorro();
  echo $animal->viver();
  echo $animal->comer("Pizza");
?>

Métodos e propriedades estáticas

Os métodos e propriedades estáticas podem ser chamados mesmo sem instanciar um objeto antes! Para definí-los como estáticos, devemos usar a palavra reservada static. Para usar um método ou propriedade estática dentro da classe, ao invés de usarmos a palavra reservada $this->, iremos usar a palavra reservada self com o operador de dois pontos duplos ::. E para acessar o valor deles, fora da classe devemos usar o nome da classe, seguido por pelo operador de dois pontos duplos ::.

<?php
  class Animal{
    public static $nome = "sem nome";

    public function __construct() {
    self::viver();
  }
    public static function viver(){
      echo "Estou VIVO!";
    }
  }

echo Animal::$nome . "<br>";
Animal::viver();
?>

Constantes dentro de classes

Sabemos que as constantes não podem ter seus valores alterados após serem declarados. Uma constante de um classe é definida com a palavra reservada const (É recomendado que constantes tenham letras maiúsculas). Para usar uma constante dentro da classe, ao invés de usarmos a palavra reservada $this->, iremos usar a palavra reservada self com o operador de dois pontos duplos ::. E para acessar o valor de uma constante de um classe, fora da classe, devemos usar o nome da classe, seguido por pelo operador de dois pontos duplos ::.

<?php
  class Animal{
    const NOME = "sem nome";
    const MSG =  "Estou VIVO!";

    public  function viver(){
      echo self::MSG;
    }
  }

  echo Animal::NOME . "<br>";
  Animal::viver();
?>

Namespaces

Namespaces são usados para resolver dois problemas, o primeiro é permitir uma melhor organização permitindo agrupar classes que trabalham juntas para realizar uma tarefa e o segundo é permitir que o mesmo nome seja usados para mais de uma classe.

Para criar um Namespace precisamos usar a palavra reservada namespace seguida pelo nome que queremos dar para o espaço. Para usar um namespace dentro de outro namespace precismos usar a palavra reservada use, e caso queiramos dar um apelido para o nome do espaço usado, podemos usar a palavra reservada as no momento em que formos usar um namespace, como em use espaco1 as E1.

Namespaces são úteis quando usamos bibliotecas de terceiros que muita vezes podem ter classes com nomes padrões que podem conflitar com o nosso cógido. Por isso criamos espaços diferentes para as bibliotecas e o nosso cógido. Desta forma, para usar uma classe de outro espaço, basta usar palavra reservada use para invocar outro espaço para dentro do nosso espaço atual, então para usar a classe externa usamos a sintaxe nomeEspaço\classeExterna.

<?php
  namespace espaco1;
  class Pessoa{
    static public $nome = "Fulano";
  }

  namespace espaco2;
  use espaco1;
  class Pessoa{
    static public $nome = "Sicrano";
  }

  echo Pessoa::$nome; //Exibe Fulano
  echo espaco1\Pessoa::$nome; //Exibe Sicrano
?>
